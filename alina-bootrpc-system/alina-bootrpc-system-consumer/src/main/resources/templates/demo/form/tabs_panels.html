HTML5 document type

Mobile device first

In Bootstrap 2, we added mobile-friendly styling to some key parts of the framework. In Bootstrap 3, we rewritten the entire framework to make it mobile-friendly from the beginning. This time, instead of simply adding some optional mobile-oriented styles, they are directly integrated into the framework's kernel. In other words, Bootstrap is mobile-first. Rather than add an extra file, mobile-specific styles are incorporated into every corner of the frame.

Typesetting and linking

Bootstrap typography and link styles set basic global styles. They are: Set background-color for the body element: #fff; Use @ font-family-base, @ font-size-base and @ line-height-base a variables as basic parameters for typesetting. Set the basic color for all links @ link-color, and underlining only when the link is in the: hover state. These styles can be found in the scaffolding.less file.

Normalize.css

To enhance consistency across browsers, we use Normalize.css, a CSS reset style library maintained by Nicolas Gallagher and Jonathan Neal.

Layout container

Bootstrap needs to wrap a .container container for the page content and the grid system. We provide two classes for this purpose. Note that these two container classes cannot be nested inside each other due to properties such as padding.

Grid system

Bootstrap provides a responsive, mobile-first streaming grid system. As the size of the screen or viewport increases, the system will automatically divide into a maximum of 12 columns. It contains easy-to-use predefined classes, and powerful mixins for generating more semantic layouts.
